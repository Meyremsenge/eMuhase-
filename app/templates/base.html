Firebase'i Bağla

Kendi Firebase projeni eMuhasebe ile birleştir

Nasıl Firebase config'i alırım? (Tıkla)
  1. Firebase Console'a git ve yeni bir proje oluştur (ya da mevcut projeni seç).
  2. Sol menüden Realtime Database'i etkinleştir. Test modunda başlayabilirsin.
  3. Sol üstte Project Settings (⚙ dişli simgesi) > General'a gel.
  4. "Your apps" bölümünden Web (</>) simgesine tıkla ve uygulamayı kaydet.
  5. Firebase sana bir firebaseConfig nesnesi gösterecek; aşağıdaki alanlara kopyala. (Ya da JSON'u olduğu gibi yapıştır.)

Veriler doğrudan kendi Firebase projende saklanır. eMuhasebe bu config'i tarayıcında lokal olarak tutar.

Güvenlik kuralları (production için önerilen) 🔒

Firebase Console → Realtime Database → Rules sekmesine aşağıdaki kuralları yapıştır. "Test mode" başlangıçta açıktır ve 30 gün sonra herkese kapanır. Üretimde mutlaka değiştir.

Sadece kendin için (en basit):

{
  "rules": {
    ".read":  "auth != null",
    ".write": "auth != null"
  }
}

Tek kullanıcılı (UID eşleşmesi):

{
  "rules": {
    ".read":  "auth != null && auth.uid == 'BURAYA_KENDI_UID'",
    ".write": "auth != null && auth.uid == 'BURAYA_KENDI_UID'"
  }
}

Test mode ({"\".read\": true, \".write\": true"}) projeni internete açık bırakır. Asla üretimde kullanma. Detaylı rehber: firebase.google.com/docs/database/security

Yapay Zeka Bağla

Google Gemini veya OpenRouter API anahtarınla AI Analiz özelliğini etkinleştir

Nasıl API anahtarı alırım? (Tıkla)
  1. Google Gemini API için (Önerilen): Google AI Studio'ya gidin, "Get API Key" butonuna tıklayarak ücretsiz AIzaSy... ile başlayan anahtarınızı oluşturun ve kopyalayın.
  2. OpenRouter için: openrouter.ai'a gidin, hesap oluşturup Keys sekmesinden sk-or-v1-... anahtarınızı alın.

Anahtarınız yalnızca tarayıcınızda (localStorage) şifreli tutulur. Backend, yapıştırdığınız anahtar formatına göre otomatik olarak doğru API'ye (Google Gemini veya OpenRouter) yönlendirme yapar.

Anahtarınız gizli kalır; hiçbir şekilde loglanmaz.
